# Decisions

- D001 **Caching: defer until performance measurement justifies it** — No caching layer (LRUCache or other library) is implemented in the MCP server or knowledge sync module. The `auto_retrieve` tool creates a fresh `GSDKnowledgeSync()` instance per call, which regenerates embeddings on each invocation. This is acceptable for current usage patterns where embedding generation is fast and call volume is low. Caching will be added only after profiling shows measurable benefit (e.g., repeated queries against the same knowledge base in a single session). When implemented, prefer the [`lru-cache`](https://www.npmjs.com/package/lru-cache) package for its well-maintained API, size limiting, and TTL support. Scope: embedding cache in `auto_retrieve`, not Qdrant search results (which are already cached by Qdrant's internal mechanisms).
